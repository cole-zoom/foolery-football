# PRD 3.4 — Eval attribution + ship gate

The eval currently records only per-week totals, so a losing week is
a number, not a diagnosis. This PRD makes lost points attributable
and defines the promotion bar for new models.

## Attribution

### Persist slot picks

`replay_week_comparison` already returns `slot_picks`
(slot → model pick + human pick); `evals/run_eval.py` just doesn't
write them. Extend the per-model cell in
`evals/results/<season>/<league_id>.json`:

```json
"gbt": {
  "predicted": 112.3,
  "actual": 98.1,
  "picks": [
    {"slot": "WR1", "model": "4046", "human": "6794",
     "model_actual": 12.3, "model_predicted": 15.1, "human_actual": 21.0}
  ]
}
```

Cell size stays small (≤ 14 slots). Old result files without `picks`
still aggregate (the field is optional) — no re-run required, but the
full attribution report needs one full re-run (~35 min).

### New metrics in `evals/aggregate.py`

Per model, over all league-weeks:

- **Ghost starts**: picks whose player has no stat row (or 0.0
  actual) that week, while the week-W roster had an eligible
  alternative with > 0 actual. Report count/week and points lost
  (best alternative − 0).
- **Benched the human's best**: weeks where the human's highest
  actual starter wasn't in the model lineup at any eligible slot —
  points difference vs the model's pick in that slot.
- **Loss decomposition**: for each losing week, split the margin into
  ghost-start points vs ranking-error points (non-ghost picks that
  underperformed the human's choice at the same slot). This tells us
  whether 3.1 (availability) or 3.2 (estimation) should get the next
  unit of effort.

Console output: one table per model; JSON in `summary_<season>.json`
under `"attribution"`.

## Ship gate

A model becomes the web default (`ComparisonView`/`ModelsDashboard`
default and CLI default) only when, on the **frozen sample**
(`evals/leagues_2025.json`, rng-seed 42 — regenerate once, commit the
league list to `evals/testdata/` as a fixture so the sample can't
silently drift):

1. Weekly win rate vs human ≥ 40% and mean season margin ≥ −75
   (context today: 27.9% / −152).
2. Over-prediction bias ≤ +5 pts/wk.
3. Backtest (`scripts/backtest-models.py`, 2025, weeks 4–18)
   startable-MAE not worse than context.
4. Seed (footballguys) league: not required to beat the human —
   required to not be the *worst* model there (guards against a model
   that games the metric on weak leagues).

Anything that clears 1–4 ships; the bar deliberately does not require
beating the median human on the first iteration (that's the milestone
goal, reached by stacking 3.1 + 3.2 + 3.3).

## Process

Every model/assembly PR must paste the before/after of
`evals/aggregate.py` (all-leagues table + attribution table) in the
PR body. The eval is the ship gate; the backtest is the guard rail;
the dashboard is the demo.

## Tests

- `league_summary`/aggregation unit-testable already — add a tiny
  synthetic results fixture under `evals/testdata/` covering: cell
  without `picks` (backward compat), a ghost start, a
  benched-best week; assert the three metrics by hand-computed
  values. Run via the existing smoke pattern (no network).

## Out of scope

- Web UI for the eval report (candidate milestone 4: a leaderboard
  page fed by `summary_<season>.json`).
- Multi-season eval sweeps (the harness takes `--season`; running
  2024 as a second held-out set is one command, just not required
  for the gate).
