# PRD 3.3 — Lineup assembly (optimal slot assignment)

Both lineup builders (the `/decisions` router and
`core/replay.py::replay_week_comparison`) fill slots **greedily in
`roster_positions` order**: each slot takes the best remaining
eligible player. That's wrong whenever a flexible slot precedes a
restrictive one — in a league listing `SUPER_FLEX` before `QB`, the
best QB lands in the flex and the QB slot gets the leftovers. The
eval shows superflex leagues ~55 pts/season worse than standard;
assembly order is a prime suspect.

## Change

New pure function in `decision_engine/core/lineup.py`:

```python
def assign_lineup(
    slots: list[str],                          # selectable slots, league order
    candidates: dict[str, ScoredCandidate],    # player_id -> scored (the shared score_cache)
    eligible: Callable[[Player, str], bool],   # player_eligible_for_slot
) -> dict[str, str | None]:                    # slot_id -> player_id
    """Maximize total final_score over one-player-per-slot assignments.

    Same bitmask DP as replay.perfect_lineup_total (slots ≤ 14,
    players sorted for determinism). Prefers the fullest lineup, then
    the highest total — an empty slot never beats a startable player.
    Ties broken by (higher final_score, lower player_id) so output is
    stable across runs.
    """
```

Callers:

- `api/routers/decisions.py::get_decisions` — replace the greedy
  accumulation loop. The per-slot *candidate lists* (what the UI
  shows per slot) are unchanged; only the `pick` per slot comes from
  `assign_lineup`.
- `core/replay.py::replay_week_comparison` — same replacement, so the
  eval and the dashboard measure the shipped behavior.

`perfect_lineup_total` stays as is (it optimizes *actual* points; this
optimizes *predicted* — same machinery, different objective) but both
share the DP helper internally.

## Semantics to preserve

- One score per player per week (shared `score_cache`) — assignment
  never re-scores.
- The `exclude_player_ids` request knob still works for single-slot
  `/decide` calls (out of scope here — those are single-slot by
  definition).
- Slots > 14 (`PERFECT_LINEUP_MAX_SLOTS`): fall back to the current
  greedy loop rather than refuse (the eval sampler already caps at
  14, but live leagues aren't sampled).

## Tests

- Superflex ordering: `("SUPER_FLEX", "QB")` with one great QB and a
  good WR → QB fills `QB`, WR fills the flex (greedy would burn the
  QB in the flex). Mirror-order league produces the identical lineup
  — assembly must be order-invariant.
- Fullness: a weak-but-eligible player beats an empty slot.
- Determinism: shuffled candidate insertion order, same lineup.
- Regression: existing decisions/comparison router tests unchanged
  for leagues where greedy was already optimal (most standard
  fixtures) — any fixture whose expected lineup changes must change
  to a strictly higher predicted total.

## Acceptance

Eval rerun on the frozen sample (existing models): superflex-league
mean margin improves; standard leagues do not regress; seed-league
totals change only upward or not at all. Report the before/after in
the PR body.
