# PRD 2.3 — CLI output

## Command

```
uv run decide \
  --user <sleeper_username> \
  --league <league_id> \
  --slot <slot> \
  [--risk <0.0–1.0>] \
  [--prefer-team <NFL_TEAM_CODE>] \
  [--avoid-team <NFL_TEAM_CODE>] \
  [--pool roster|waivers|both] \
  [--limit <N>] \
  [--season <year>] \
  [--week <n>] \
  [--model <name>]
```

Defaults: `--risk 0.5`, `--pool roster`, `--limit 10`, `--model naive`.
`--season` / `--week` override the live `/v1/state/nfl` lookup (used
for replaying past weeks or testing against fixtures).

## Output shape

Plain-text table, sorted by `score` desc:

```
Snapshot: data/snapshots/2026-09-15/  (season 2026, week 2)
League:   "Sunday Funday" (1234567890), scoring: PPR (rec=1.0)
User:     cole (user_id 987654321)
Slot:     FLEX  Risk: 0.30  Pool: roster  Model: naive

Rank  Player              Team  Pos  Mean   Var    Score   Notes
   1  Justin Jefferson    MIN   WR   18.4   4.1    16.4
   2  Tyreek Hill         MIA   WR   16.9   6.8    13.5    questionable
   3  Bijan Robinson      ATL   RB   14.2   3.5    12.4
   4  Sam LaPorta         DET   TE   11.8   2.9    10.6    +10% DET preference
   ...
```

- `Score` = the risk-adjusted score from the scoring model (with
  any team preference multipliers applied).
- `Notes` surfaces `injury_status`, "no historical data", team
  preference adjustments, bye weeks if surfaced by the model, etc.
- The header includes the snapshot date, season/week, league
  scoring summary, and the model used — so the buddies can compare
  runs without ambiguity.

## Exit codes

- `0` — printed a ranked list (even of length 0 if the eligible pool was empty).
- `1` — user input validation error (unknown user, league mismatch, bad slot, bad risk).
- `2` — runtime error (Sleeper API down, snapshot missing, schema mismatch, malformed snapshot).

## Library mode

The `decide` entrypoint is a thin wrapper over `core.pipeline.run`,
which returns a list of `PlayerScore`. Anyone wiring this into a
future web UI imports `core.pipeline.run` directly and never
touches the CLI.

## Out of scope

- JSON output mode. Easy to add (`--format json`) when something
  consumes it. Until then, the buddies read tables.
- Color / TUI. Plain text.
- Pagination. `--limit` is sufficient.
- Saving recommendations to disk for later. Each invocation is
  self-contained.
