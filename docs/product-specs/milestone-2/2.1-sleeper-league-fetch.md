# PRD 2.1 — Sleeper league fetch

The decision engine fetches league context live from Sleeper on
every CLI invocation. League state (rosters, matchups) changes
frequently and is small (a few KB), so caching isn't worth the
complexity.

## Endpoints used

| Purpose | Endpoint |
| -- | -- |
| Username → user_id | `GET /v1/user/<username>` |
| User's leagues for a season | `GET /v1/user/<user_id>/leagues/nfl/<season>` |
| League settings + scoring | `GET /v1/league/<league_id>` |
| League rosters (who owns whom) | `GET /v1/league/<league_id>/rosters` |
| Current week matchups | `GET /v1/league/<league_id>/matchups/<week>` |
| League users (for display) | `GET /v1/league/<league_id>/users` |

See [`docs/references/sleeper-api.md`](../../references/sleeper-api.md)
for response shapes.

## Resolution flow

1. `GET /v1/user/<username>` → `user_id`. 404 → exit `1` with
   "Unknown Sleeper username."
2. `GET /v1/user/<user_id>/leagues/nfl/<season>` → list. The
   `--league` flag must match one of these IDs. If not: print the
   user's available leagues with names + IDs, exit `1`.
3. `GET /v1/league/<league_id>` → keep `roster_positions` and
   `scoring_settings`.
4. `GET /v1/league/<league_id>/rosters` → find the entry whose
   `owner_id` matches our `user_id`. The `players` field on that
   roster is the user's full roster (start + bench).
5. Free-agent pool: `{all NFL players in snapshot's players.json} − {players in any roster on the league}`.

The `season` used in step 2 comes from `/v1/state/nfl` (the same
call the stats-loader uses), unless overridden with `--season`.

## Slot eligibility

The league's `roster_positions` field is a list like
`["QB", "RB", "RB", "WR", "WR", "TE", "FLEX", "K", "DEF", "BN", "BN", ...]`.

For a given `--slot`:

- Exact-match positions (`QB`, `RB`, `WR`, `TE`, `K`, `DEF`) accept
  players whose `fantasy_positions` includes that position.
- `FLEX` accepts RB / WR / TE by default. Some leagues use
  `WRRB_FLEX`, `WRT_FLEX`, `SUPER_FLEX`, etc. — these are decoded
  in [`docs/references/fantasy-glossary.md`](../../references/fantasy-glossary.md)
  via a flex-eligibility map. Any unknown slot aborts with
  "unsupported slot — add it to the flex map."
- `BN` / `IR` / `TAXI` are not selectable slots; specifying them
  errors with exit `1`.

This is the place where leagues with non-default position scopes
(e.g. defensive-only) get supported: by adding entries to the
eligibility map. No code path treats QB/RB/WR/TE as special-cased;
they're just the default `roster_positions` shape.

## Failure modes

- Any Sleeper call retries with exponential backoff (3 attempts)
  then aborts with exit `2`.
- Snapshot missing or empty → exit `2` with "no snapshots found,
  run `stats-loader update` first."
- Snapshot `schema_version` newer than supported → exit `2` with a
  clear "decision-engine is older than your snapshot, upgrade or
  pin" message.
- League ID doesn't match any of the user's leagues → list
  available leagues, exit `1`. **Never** silently fall through.
- User's roster not found in `/rosters` (shouldn't happen if
  `/leagues/nfl/<season>` returned it) → exit `2`, this is a bug.
